# Decision Records

This folder is the canonical home for shared decision records.

## Structure

- `docs/decisions/adr/` — architecture decision records with explicit context, options, decision, and consequences
- `docs/decisions/` (root) — broader domain, data, regulatory, and implementation decision notes that are not strict ADRs

## Conventions

- Use `docs/decisions/adr/` when the document records a stable architectural choice.
- Use `docs/decisions/` root for supporting decision notes, mappings, and domain-specific implementation decisions.
- Prefer linking to files here from specs, audits, and roadmaps instead of creating parallel decision folders elsewhere.
