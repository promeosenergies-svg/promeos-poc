# Audit Customer Experience (CX) PROMEOS

> **CX ≠ CS ≠ UX** :
> - **UX** = qualité interaction écran (touch, lisibilité, animations)
> - **CS** = succès du client à atteindre sa valeur sur la durée
> - **CX** = **émotion + perception** du client à chaque touchpoint produit
>
> **Étape 11** du plan séquentiel utilisateur étendu.

---

## 1. TL;DR

1. **Voix produit Sol cohérente** ✅ : grammaire éditoriale (kicker + titre narratif + footer source/confiance) appliquée Cockpit dual. Triptyque typographique Fraunces/DM Sans/JetBrains Mono = signal premium.
2. **Touchpoints critiques identifiés** : login → cockpit → drill-down → action → retour cockpit. Boucle CX courte si parcours fluide.
3. **Doctrine §1 promesse** : "OS énergétique vivant" + "magazine spécialisé". Réalité actuelle : ✅ cockpit dual livré, ⚠️ "vivant" partiel (Chantier α moteur événements proactif backlog).
4. **Émotion attendue** : sérénité ("le produit pense pour moi") + autorité ("sources citées") + agentivité ("je peux agir maintenant"). État actuel : sérénité ✅, autorité ✅, agentivité ⚠️ (action center bien mais pas push proactif).
5. **Trois trous CX P1** : (1) pas de tone-of-voice consistance dans empty/error states, (2) pas de moment "wow" aux étapes clés (premier insight découvert, première anomalie résolue), (3) pas de personnalisation premier prénom.

```

Émotion : Espoir → Confiance/Doute → Curiosité → Engagement → Satisfaction → Sérénité
```

## 3. Voix produit Sol — analyse cohérence

### 3.1 Grammaire éditoriale (doctrine §5)

Doctrine §5 fixe la structure invariante :
```
[KICKER]            ← contexte
TITRE NARRATIF      ← Fraunces, voix produit
Narrative 2-3 lignes ← intro qui raconte, sourcée
[KPI 1] [KPI 2] [KPI 3]  ← 3 max, tooltips
CETTE SEMAINE CHEZ VOUS  ← week-cards sémantiques
[À regarder] [À faire] [Bonne nouvelle]
[FOOTER : SOURCE · CONFIANCE · MIS À JOUR]
```

### 3.2 Conformité par page

| Page | Kicker | Titre narratif | Footer source/confiance | CX vibe |
|---|---|---|---|---|
| /cockpit/jour | ✅ SolKickerWithSwitch | ✅ Fraunces titre | ✅ SolBriefingFooter | OK |
| /cockpit/strategique | ✅ idem | ✅ idem | ✅ idem | OK |
| /conformite | ⚠️ partial | ⚠️ partial | ⚠️ partial | P1 |
| /bill-intel | ❓ (audit out of scope) | — | — | À vérifier |
| Empty states (AsyncState) | ❌ generic message | ❌ pas de Fraunces | ❌ pas de footer | **P1 CX** |
| Error states (ErrorBoundary) | ❌ generic icone | ❌ generic title | ❌ pas de signal Sol | **P1 CX** |

→ **Issue P1 CX** : la voix Sol s'arrête aux pages "happy path". Empty states + error states cassent l'immersion (passage du mode "magazine premium" au mode "outil technique générique").

### 3.3 Recommandation P1 — Sol-ifier les states limites

```jsx
// ❌ Avant (AsyncState empty générique)
<AsyncState empty={!data?.length} emptyMessage="Aucun site trouvé">

// ✅ Après (Sol-ifié)
<AsyncState
  empty={!data?.length}
  emptyMessage="Vous n'avez pas encore connecté de sites"
  emptyAction={<Button>Importer mes sites depuis Sirene</Button>}
  emptyKicker="ÉTAPE SUIVANTE"
  emptyIllustration={<EmptySitesIllustration />}
>
```

## 4. Émotion premium — signaux qualité perçue

### 4.1 Triptyque typographique (Sol)

| Font | Usage | Présent ? |
|---|---|---|
| **Fraunces** (serif) | Titres narratifs | ✅ vérifié frontend/index.html + tokens.css |
| **DM Sans** (sans-serif) | Corps + UI | ✅ idem |
| **JetBrains Mono** | Kickers + KPIs tabular-nums | ✅ idem |

→ **Signal premium fort** ✅ — différenciant face à Inter B2B SaaS générique.

### 4.2 Color-life tints

7 tints couleur (blue/emerald/indigo/amber/violet/cyan/slate) appliqués à 12 propriétés Tailwind chaque. Cohérence rail / panel / header.

→ **Signal qualité fort** ✅.

### 4.3 Glass UI

`backdrop-blur-sm` + `bg-white/80` pour rail/panel/header. Effet glass moderne.

→ **Signal moderne** ✅ mais subtil sur fond blanc.

### 4.4 Issues P1 CX premium

#### P1.1 — Login page basique

[LoginPage.jsx] — non audité ici mais probablement page login standard sans signal Sol premium (kicker, narrative, illustration).

→ **Premier touchpoint = première impression**. Si login basique, le user n'attend pas un produit premium.

**Recommandation** : Login Sol-ifié = bandeau gradient + slogan ("L'OS énergétique qui pense pour vous") + illustration rail/cockpit en fond.

#### P1.2 — Loading states sans signal Sol

`SkeletonBlock` standard. Aucune mention "PROMEOS calcule..." ou kicker pendant le chargement.

**Recommandation** : skeleton "smart" qui annonce ce que PROMEOS fait :
- "Analyse de votre patrimoine..." (cockpit loading)
- "Audit de vos factures..." (bill-intel loading)
- "Score conformité en cours..." (conformite loading)

→ **CX boost** : transforme l'attente en anticipation.

## 5. Personnalisation — la voix qui s'adresse

### 5.1 État actuel

L'app n'utilise jamais le **prénom** de l'utilisateur dans la voix produit. Ex : Briefing du jour dit "Voici votre journée" mais pas "Bonjour Marie, voici votre journée".

### 5.2 Personnalisation simple disponible

- `auth.user.prenom` ou `auth.user.nom` est dans AuthContext
- ROLE_LABELS pour mapper le rôle
- Date/jour pour saluer ("Bonjour" vs "Bonsoir")

### 5.3 Recommandation P1 — Pers attendrissante

```jsx
// Cockpit Briefing du jour header
"Bonjour Marie, voici votre journée énergie."
//   ^prénom    ^contexte temporel
```

Avec fallback : "Bonjour, voici votre journée énergie." si prénom absent.

→ **Effort minimal** + **impact CX émotionnel** mesurable (sentiment d'attention personnalisée).

## 6. Moments "Wow" — à instrumenter

### 6.1 Moments potentiels (doctrine §1 magazine spécialisé)

| Moment | Signal CX attendu |
|---|---|
| Première anomalie billing détectée | Toast premium "🎯 PROMEOS a trouvé 47 € d'écart sur votre facture février." |
| Première action complétée | Confetti subtil + "Bravo, action n°1 complétée." |
| Premier site importé | "Site ajouté à votre patrimoine. Analyse démarrée." |
| Score conformité atteint > 80 % | "Vous êtes au-dessus de la moyenne du secteur." |
| Économie cumulée 1 k€ / 10 k€ / 100 k€ | "Vous avez économisé X € grâce à PROMEOS." (doctrine §4 promesse mesurable) |

### 6.2 État actuel : 0/5 moments instrumentés

Aucun de ces signaux "wow" n'est actuellement présent. → **Trou CX majeur P1**.

### 6.3 Recommandation P1 — Moments wow toast premium

Ajouter un système de moments "wow" :
- Trigger backend : événement métier (anomalie détectée, action complétée, milestone)
- Trigger frontend : toast premium avec voix Sol (Fraunces titre + DM Sans corps)
- Persistence : éventuellement timeline des "wows" du compte (gamification douce)
