Sprint 7.1 changelog. Zero breaking change, additive only.

## 1. Import idempotent

- `POST /api/billing/import-csv` now computes SHA-256 of CSV content
- Re-upload of same file (same org + same hash) returns `{"status": "already_imported"}`
- New model `BillingImportBatch` tracks every import: filename, content_hash, rows stats
- New endpoint `GET /api/billing/import/batches` lists all import batches

## 2. Deduplication

- `EnergyInvoice` has `UniqueConstraint("site_id", "invoice_number", "period_start", "period_end")`
- Row-level dedup during CSV import: existing invoice_number â†’ skipped (counted separately)

## 3. Price reference â€” single source of truth

New function `get_reference_price(db, site_id, energy_type, period_start, period_end)`:

| Priority | Source | Example |
|----------|--------|---------|
| 1 | `EnergyContract.price_ref_eur_per_kwh` (active, covering period) | `(0.22, "contract:5")` |
| 2 | `SiteTariffProfile.price_ref_eur_per_kwh` | `(0.19, "site_tariff_profile")` |
| 3 | Fallback: 0.18 elec / 0.09 gaz | `(0.18, "default_elec")` |

`shadow_billing_simple` now accepts optional `db` param to resolve price via this chain.
Response includes `ref_price_source` field.

## 4. Insight workflow

New enum `InsightStatus`: OPEN, ACK, RESOLVED, FALSE_POSITIVE.

New columns on `BillingInsight`: `insight_status`, `owner`, `notes`.

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| PATCH | `/api/billing/insights/{id}` | Update status/owner/notes |
| POST | `/api/billing/insights/{id}/resolve` | Shortcut â†’ RESOLVED |
| GET | `/api/billing/insights?status=open` | Filter by status |

`GET /api/billing/insights` response now includes `insight_status`, `owner`, `notes`.

## 5. Proof & explainability

All 10 anomaly rules (R1-R10) now include in `metrics`:
- `inputs`: standardized dict with `period_start`, `period_end`, `energy_kwh`, `total_eur`, `implied_price`, `ref_price`
- `threshold_*`: the threshold that triggered the rule
- `delta_calculated_*`: the computed delta value

## 6. Cross-link conso <-> facture

R9 (consumption_spike) now includes:
- `recommended_actions`: link to `/diagnostic-conso?site_id=X&from=Y&to=Z`
- `metrics.cross_link`: `{"module": "diagnostic-conso", "site_id": ..., "date_range": {...}}`
